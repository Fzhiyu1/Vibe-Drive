# 测试08：动态氛围场景（开放题）

## 难度：⭐⭐⭐⭐⭐+

## 需求描述

创建一个**动态变化的车内氛围场景**，能够根据不同的"情绪/场景模式"自动切换氛围效果。

## 核心要求

1. 基于 07-车内场景 的基础结构
2. 实现至少 **3 种氛围模式**，例如：
   - 放松模式
   - 专注模式
   - 活力模式
3. 每种模式应有**明显不同**的视觉表现
4. 模式切换时有**过渡动画**

## 开放点（自由发挥）

以下方面**不做具体限制**，由 AI 自行设计：

- 每种模式的具体颜色、动画效果
- 过渡动画的实现方式
- 是否添加额外的视觉元素（粒子、光晕、雾气等）
- 是否添加音效或其他交互
- UI 设计风格

## 评估维度

| 维度 | 说明 |
|------|------|
| 创意性 | 模式设计是否有新意 |
| 视觉效果 | 氛围表现是否有感染力 |
| 技术实现 | 代码质量、动画流畅度 |
| 完整度 | 功能是否完整可用 |

## 输出格式

- 单个 HTML 文件

## 验收标准

- [ ] 至少 3 种可切换的氛围模式
- [ ] 模式之间有过渡效果
- [ ] 视觉效果有明显差异
- [ ] 代码可运行
